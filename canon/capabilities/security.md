# Security

## Purpose

Security review for game projects, not just generic applications.

## Responsibilities

- auth and access control review
- trust boundaries
- state manipulation risks
- exploit surfaces
- economy abuse
- griefing vectors
- bot profitability
- incident containment thinking

## Key Question

How does this get abused by a smart attacker or a selfish player?
