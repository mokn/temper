# Audit

## Purpose

Inspect the repo for doctrine drift, architectural erosion, or silent inconsistency.

## Responsibilities

- source-of-truth drift detection
- command and workflow drift
- architecture and folder-shape drift
- stale config or dead-path surfacing

## Key Question

What has quietly become inconsistent, unsafe, or hard to reason about?
